# Code Rules

> Generated by BobTheDocker · 2026-06-27

## Style Rules

- Use PEP 8 conventions: snake_case for variables/functions, PascalCase for classes, UPPER_CASE for constants.
- All Python code must include type hints for function signatures and class attributes.
- Pydantic models (v2) must extend `BaseModel`, SQLAlchemy models must extend `DeclarativeBase`.
- API routes must be async if they perform I/O (database, file, network). Use `async def` in FastAPI path operations.
- Use FastAPI’s dependency injection for database sessions (`Depends(get_db)`) and other shared logic.
- Organise code: put models in `models.py`, Pydantic schemas in `schemas.py`, routers in `routers/` directory, and business logic in separate service modules.
- Use `APIRouter` for grouping related endpoints, not app decoration directly in `main.py` (except root route).
- Keep response models explicit: always set `response_model` in route decorators.
- Docstrings are mandatory for all public modules, classes, and functions, using Google style. Include Args, Returns, and Raises sections.

## Required Practices

- **Every PR must include tests**: add/update unit and integration tests in `test_*.py` files using `pytest` and `httpx.AsyncClient`. Cover happy path, validation errors, edge cases, and error handling.
- **Error handling**: raise `HTTPException` with appropriate status codes (4xx client errors, 5xx server errors). Use try/except around database operations; catch `IntegrityError` and `NoResultFound` specifically.
- **Logging**: use the Python `logging` module with a named logger (`logger = logging.getLogger(__name__)`). Log at INFO for normal operations, WARNING for soft errors, ERROR for exceptions. Never use `print()`.
- All new endpoints must be documented in README.md and OpenAPI docs (via docstrings, `summary`, `description`, `tags`).
- Use environment variables for configuration (e.g., database URL) via `pydantic.BaseSettings` or `os.getenv`, not hardcoded strings.

## Forbidden Patterns

- **No hardcoded secrets, credentials, or API keys** in the codebase. Use `.env` + `.env.example`.
- **Never commit `.db` files, `__pycache__/`, `.env`, or IDE folders.** They must be in `.gitignore`.
- **No raw SQL strings** – use SQLAlchemy ORM queries only.
- **No bare `except:`** – always catch specific exceptions.
- **No commented-out code** – remove dead code before submission.
- **No unused imports or variables** – remove them.
- **No `print()` or `sys.stderr.write()` for debugging** – use logging.
- **No synchronous blocking calls in async routes** except via `run_in_executor`.
- **No star imports** (`from module import *`).

## PR Checklist

Before calling `submit_pr`, ensure all the following are true:

- [ ] All tests pass locally (`pytest -v`).
- [ ] No new flake8/pylint/black violations; code is formatted with `black` (if used).
- [ ] Every new endpoint/function is covered by tests.
- [ ] Proper `HTTPException` raising with meaningful detail messages.
- [ ] Logging statements are present at appropriate levels, no `print()`.
- [ ] Database operations use dependency-injected sessions; sessions are closed correctly.
- [ ] Response models match the returned data exactly.
- [ ] README is updated if endpoint signature or usage changes.
- [ ] No sensitive data or temporary files committed.
- [ ] The PR has a descriptive title and links to the related issue.
