# Code Rules

> Generated by BobTheDocker · 2026-06-27

## Code Style and Patterns
- Follow PEP 8 naming, indentation, and line length.
- All public functions/methods must have type hints.
- FastAPI endpoint functions are `async`; use dependency injection for DB sessions and configuration.
- SQLAlchemy uses 2.0 style: `select()` statements, `session.execute()`, and `result.scalars()`. Models inherit from `DeclarativeBase`.
- Pydantic models (v2) use `model_validate`, `model_dump()`. Schema files define request/response shapes.
- Project layout: `main.py` (app, routes), `models.py` (SQLAlchemy), `schemas.py` (Pydantic), `test_main.py` (tests).

## Required Practices
- **Testing:** Every new endpoint or modified logic must have covering tests. Tests include:
  - Happy path (valid input → correct status/data).
  - Validation errors (missing required field, negative price → 422/400).
  - Not‑found and conflict scenarios.
  - Edge cases (boundary values, empty lists).
- **Error Handling:** Use `fastapi.HTTPException` with accurate status codes (`400`, `404`, `422`). Never return raw exceptions. Response error body follows `{"detail": str}` pattern.
- **Logging:** Use Python `logging` (not `print`). Log at least:
  - `INFO`: product created, updated, deleted.
  - `WARNING`: soft failures, fallbacks.
  - `ERROR`: unhandled exceptions (via middleware/exception handler).
- **Database Actions:** Explicitly commit on success, rollback on exception within a transaction context.

## Forbidden Patterns
- **No `.db` file** in commits; ensure `grocery_store.db` is in `.gitignore`.
- **No raw SQL strings**; use SQLAlchemy expression language or ORM.
- **Never swallow exceptions:** No bare `except:`, no `pass` in except blocks. Always log or re‑raise.
- **No global mutable state** that persists across requests.
- **No `import *`**; import only what is needed.
- **No `print()`**; replace with `logger.info()`.
- **No hardcoded secrets** or connection strings.
- **No unused imports or variables** left in final code.

## PR Checklist
Before calling `submit_pr`, confirm all items:
- [ ] All new routes/features have corresponding tests in `test_main.py` (or dedicated test file).
- [ ] `pytest -v` runs without failures.
- [ ] Code passes `mypy` (if type‑checked) and `black` (if formatted).
- [ ] Logging uses correct level, no `print` left.
- [ ] Error responses are consistent and informative.
- [ ] Database file (`*.db`) is not included in the changeset.
- [ ] `README.md` updated if API signatures, environment variables, or setup instructions changed.
- [ ] Pull request includes only relevant changes; no unrelated files or whitespace noise.
