# Code Rules

> Generated by BobTheDocker · 2026-06-27

## Language & Style (Python / FastAPI / SQLAlchemy + Pydantic)

- All Python code must follow **PEP8** conventions (snake_case for variables/functions, CamelCase for classes). Use **type hints** on all function signatures and class attributes.
- **Pydantic schemas** (in `schemas.py`) must mirror the SQLAlchemy models (in `models.py`). No request/response models defined inline in route handlers.
- **SQLAlchemy models** use the declarative base from `models.py`. All table columns must include appropriate types, constraints, and defaults as described in the README schema.
- API route implementations belong in `main.py` unless a dedicated routers module is created (must be reviewed).
- Static assets (`static/`) are served automatically by FastAPI; do not introduce server‑side templates.

## Mandatory Testing

- **Tests are written with `pytest` and `httpx`** (`TestClient` from `fastapi.testclient`). Every new or modified endpoint **must** have corresponding test coverage (create, read, update, delete, filter, error cases).
- Run the full test suite and ensure **all tests pass** before submitting a PR:
  ```bash
  pytest -v
  ```
- Test files must be named `test_*.py`. Place endpoint tests in `test_main.py`; ensure the test database (`grocery_store.db`) is either mocked or recreated in a dedicated test session (do not alter production data).
- **Do not introduce flaky tests** (depending on execution order or external state).

## Structure & Separation

- **Database models** → `models.py` only. **Pydantic schemas** → `schemas.py` only. If new entities are added, define both.
- **Route handlers** → `main.py` (or a dedicated `routers/` module if approved). Keep business logic out of route handlers; use a service layer if complexity grows, but do not create it without explicit justification.
- **No direct SQL** – always use the SQLAlchemy session/ORM to query the database.
- **Schema validation** must be done entirely with Pydantic (e.g., `price > 0` enforced by the schema, not by manual checks).

## Forbidden Patterns

- ❌ Raw SQL strings (`engine.execute("SELECT ...")`) – use `db.query(Model)`.
- ❌ Returning SQLAlchemy model instances directly in responses – always convert through a Pydantic schema.
- ❌ Hard‑coding URL paths; use FastAPI’s `url_for` or relative paths.
- ❌ Changing the `grocery_store.db` schema without an Alembic migration (not present) – if manual schema changes are needed, document the migration steps in the PR description.
- ❌ Bypassing type hints or ignoring Pylance/mypy errors (if type checker used locally, ensure no new errors).

## PR Checklist (every item must be satisfied)

- [ ] `pytest -v` passes with zero failures.
- [ ] All new endpoints and edge cases are covered in `test_main.py`.
- [ ] Any new model/field added in `models.py` is mirrored in `schemas.py` (including validators).
- [ ] API documentation (`/docs`) renders without errors when running `uvicorn main:app --reload`.
- [ ] The README endpoint table is updated if public routes change (new or modified).
- [ ] No stray debug `print()` statements or commented‑out code.
- [ ] Static files (`static/`) are not injected with server‑side state; they remain static.
