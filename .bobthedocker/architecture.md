# Architecture Reference: Grocery Store API

## Tech Stack and Key Dependencies

- **Runtime**: Python 3.x  
- **Web Framework**: FastAPI 0.104.1  
- **ASGI Server**: Uvicorn 0.24.0  
- **ORM / Database**: SQLAlchemy 2.0.23 with SQLite (`grocery_store.db` file)  
- **Data Validation**: Pydantic 2.5.2  
- **Testing**: pytest 7.4.3 with httpx 0.25.2 (async HTTP client for FastAPI test client)  
- **Frontend**: Static HTML/CSS/JS (no framework), served from `static/` directory

## Key Directories and Files

- **`main.py`**  
  FastAPI application definition and route handlers for the product API (CRUD endpoints). Entry point for Uvicorn.

- **`models.py`**  
  SQLAlchemy ORM models defining the `products` table schema. Maps to the SQLite database.

- **`schemas.py`**  
  Pydantic models used for request/response serialisation and validation (ProductCreate, ProductUpdate, ProductResponse).

- **`hello.py`**  
  Standalone module (possible utility or legacy) — not directly referenced in the API endpoints provided in README.

- **`test_main.py`**  
  Integration tests for the product endpoints using FastAPI’s `TestClient` (backed by httpx). Covers CRUD operations, filtering, error cases.

- **`test_hello.py`**  
  Unit tests for `hello.py`.

- **`static/`**  
  - `index.html`: Frontend UI shell.  
  - `js/app.js`: Client-side logic, likely fetching from the API.  
  - `css/styles.css`: Styling for the frontend.  
  *Note: The README does not document the frontend, but the static files are present and presumably served by the app.*

- **`grocery_store.db`**  
  SQLite database file (auto-created/populated by SQLAlchemy). Stores product records.

- **`requirements.txt`**  
  Pinned dependencies for pip installation.

- **`.gitignore`**  
  Standard ignore rules for Python, venv, and database files.

- **`README.md`**  
  Project documentation covering features, schema, endpoints, and local setup steps.

## Entry Points

1. **Backend server**:  
   Module `main` exposes the FastAPI `app` instance.  
   Run with: `uvicorn main:app --reload`

2. **API documentation**:  
   Interactive docs at `http://localhost:8000/docs` (Swagger) and `http://localhost:8000/redoc`.

3. **Frontend**:  
   Served as static files (probably at `http://localhost:8000/` root if configured in `main.py` to mount `static/index.html`).

## Test Framework and How to Run

- All tests use **pytest**.  
- Test files: `test_main.py` (API integration) and `test_hello.py`.  
- Run entire suite with:  
  ```bash
  pytest -v
  ```  
- The test client is `httpx.AsyncClient` (via `TestClient`), allowing both sync and async test patterns.

## Build/Run Commands

1. Set up virtual environment (optional) and install dependencies:  
   ```bash
   pip install -r requirements.txt
   ```

2. Start development server:  
   ```bash
   uvicorn main:app --reload
   ```

3. Access API at `http://localhost:8000`.

## Notable Patterns and Conventions

- **Layered architecture**: Separation of concerns with `models.py` (DB layer), `schemas.py` (contracts), and `main.py` (routes).  
- **CRUD endpoint design**: Standard RESTful pattern for products, including partial updates via `PUT` (reads existing record, applies changes, writes back).  
- **Synchronous SQLAlchemy**: Likely uses synchronous sessions (common with SQLite). No async engine observed from file contents.  
- **File‑based SQLite**: Database stored as a local file, suitable for development.  
- **Static frontend**: Minimalist SPA built with vanilla JS, served alongside the API, creating a monolith deployment.  
- **Test conventions**: Tests use `pytest` fixtures and `httpx` client to make real HTTP requests against the app, simulating full-stack interaction.
