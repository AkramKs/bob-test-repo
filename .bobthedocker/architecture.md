# Architecture Reference: Grocery Store API

## Tech Stack & Key Dependencies
- **Python 3.x** with **FastAPI** (0.104.1) as the web framework
- **SQLAlchemy** (2.0.23) ORM + **SQLite** (`grocery_store.db`) for persistence
- **Pydantic** (2.5.2) for request/response validation
- **Uvicorn** (0.24.0) as ASGI server
- Testing: **pytest** (7.4.3) and **httpx** (0.25.2) for async HTTP client in tests

## Key Directories & Files
- **Root** – application source code, database file, and tests:
  - `main.py` – FastAPI application instance and API route definitions (CRUD)
  - `models.py` – SQLAlchemy model for `Product`
  - `schemas.py` – Pydantic schemas for validation and serialization
  - `hello.py` – a simple route/health check module
  - `grocery_store.db` – SQLite database file (auto‑managed by SQLAlchemy)
  - `test_main.py` – integration tests for product endpoints
  - `test_hello.py` – tests for the `hello` module
  - `requirements.txt` – pinned dependencies
- **`static/`** – frontend assets:
  - `index.html`, `js/app.js`, `css/styles.css` – minimal UI for consuming the API

## Entry Points
- **Application**: `main.py` exposes the FastAPI `app` object  
  Accessed via: `uvicorn main:app --host <host> --port <port>`
- **API documentation** generated automatically at `/docs` (Swagger UI) and `/redoc`

## Testing
- **Framework**: pytest
- **Test runner command**: `pytest -v`
- **Test style**: integration tests using `httpx.AsyncClient` against the FastAPI app (likely via `TestClient` or direct ASGI transport)
- Test files are co‑located with source: `test_main.py` and `test_hello.py`

## Build / Run Commands
1. Install dependencies:  
   `pip install -r requirements.txt`
2. Start development server with auto‑reload:  
   `uvicorn main:app --reload`
3. Run tests:  
   `pytest -v`

## Notable Patterns & Conventions
- **CRUD RESTful design**: endpoints follow resource‑oriented naming (`/products/`, `/products/{id}`)
- **Filtering via query parameters**: `GET /products/` accepts `category` and `is_available`
- **Partial updates**: `PUT` endpoint supports updating only provided fields
- **Schema‑first validation**: Pydantic models define request/response shapes, default values (e.g., `category="Other"`, `unit="piece"`)
- **SQLAlchemy session management**: likely uses dependency injection (e.g., `get_db`) to handle database sessions per request
- **Auto‑generated API docs**: FastAPI’s OpenAPI integration enables interactive documentation out‑of‑the‑box
- **Co‑located static files**: the API serves its own minimal frontend from `static/`
